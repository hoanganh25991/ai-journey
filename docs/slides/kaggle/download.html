Deeper divenotes

Deeper dive

quota

Quota economics

Prototype on CPU / 1% sample → flip GPU for real fit / Trainer.train() → save ckpt → turn GPU off. Interactive sessions count while the kernel is on.

metric

Metric alignment

Log-loss punishes confident wrongs; accuracy only cares about argmax. Implement a local CV that mirrors the host metric before submitting.

CV

Public NB as curriculum

Fork the simplest high scorer, re-run end-to-end, change one lever. Blind stacking of 10 notebooks usually overfits the public LB.

11Decision guide
Case Studylab

Case Study

Dog-breed image competition: ~10k images, 120 classes, scored by multi-class log-loss.

inputs

Inputs

competition train folder + labels CSV; Hub vision backbone; notebook with GPU enabled only for training cells.

steps

Steps

CPU EDA + 1% sample sanity check → enable GPU → fine-tune EfficientNet-B0 (head LR 1e-3, then full 1e-4), batch 32, 5 epochs → local stratified CV log-loss → write submission.csv → download checkpoint Dataset for the lab demo.

output

Output

public LB log-loss competitive with a simple baseline; artifact Dataset holds weights + label map + image size/normalize config.

check

Verify

nvidia-smi before long runs; GPU off while debugging plots; local metric matches host log-loss (not only accuracy); private-LB risk if you overfit public notebooks blindly.

Pitfallswatch

Easy ways to waste the workshop

quota

Burning GPU on debug

Print-debug on CPU first; tiny subsets until the loop runs. Verify accelerator with nvidia-smi.

metric

Ignoring the metric

Optimizing accuracy when the score is log-loss.

copy

Copy-paste without understanding

Public notebooks win until the private leaderboard — learn why.

export

No offline export plan

Session dies with the kernel; save artifacts deliberately. Fit scalers on train only.
