PyTorchideas

Key ideas

batch

Epoch vs batch

Epoch = one pass over all data; batch = chunk processed per step.

four

Four steps

forward → loss → backward() → step(). Always zero_grad().

lr

Optimizer + LR

Adam/SGD; LR too high → jumps; too low → crawls.

Codeskeleton

The batch loop

for epoch in range(EPOCHS): for x, y in dataloader: opt.zero_grad() logits = model(x) # forward loss = loss_fn(logits, y) loss.backward() # gradients opt.step() # update W
remember

zero_grad()

Without clearing gradients, updates accumulate across batches and training goes wild.
