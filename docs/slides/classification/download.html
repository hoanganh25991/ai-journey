Key ideas

head

Classification head

For d-dim embed and C classes: logits = Wx + b → softmax → argmax.

loss

Cross-entropy

Multi-class CE = −log p_y. True class at 0.81 → loss ≈0.21; at 0.05 → ≈3.0 — wrong & confident is expensive.

eval

Beyond accuracy

Precision, recall, F1 — especially when classes are rare or skewed.

Picturemetrics

Precision, recall, F1 from the confusion matrix

P = TP/(TP+FP) · R = TP/(TP+FN) · F1 = 2·P·R/(P+R). On skewed data, dig into the rare-class row — not just overall accuracy.

Balanceeval

Skew, calibration, operating points

skew

95% majority trap

Always-predict-majority hits 0.95 accuracy with 0 minority recall. Prefer class weights, oversampling, or macro-F1.

calib

Rank ≠ probability

argmax can be right while “90%” is only right 70% of the time. Temperature-scale on val when scores are risk inputs.

PR

Tune the cutoff

Binary spam: try threshold 0.3 for recall. Plot the val PR curve; pick the cost trade-off, not default 0.5.

Deeper divenotes

Deeper dive

logits

Unbounded → probs

Softmax never hits exact 0/1, but near-zero true-class probs explode CE. Label smoothing (ε≈0.1) softens targets and often helps calibration.

binary

One logit vs two

Sigmoid on z ≈ softmax on [0, z]. Prefer one logit + BCE-with-logits for binary; C logits + CE for multi-class.

heads

Multi-label ≠ multi-class

Softmax forces mutual exclusion. Co-occurring tags need independent sigmoids + BCE — otherwise politics & economy collapse.

15classification-case.png
Case Studylab

Case Study

Support-ticket triage for a SaaS inbox with three intents: billing, bug, how-to (~12 000 labeled tickets, ~70% how-to).

inputs

Inputs

ticket subject + body (truncated to 256 tokens), integer labels 0..2, stratified 80/10/10 split; class weights ≈ [1.4, 2.1, 0.7] because bugs are rare.

steps

Steps

load a small Hub encoder → freeze backbone for 2 epochs (head LR 1e-3) → unfreeze with AdamW 2e-5 for 3 more → early-stop on macro-F1 (not accuracy).

output

Output

checkpoint with val macro-F1 ≈ 0.78, accuracy ≈ 0.88; confusion matrix shows bug recall ≈ 0.71 — still the weak class.

check

Verify

(1) majority baseline accuracy (~0.70) so 0.88 is real lift; (2) per-class recall, especially bug; (3) no duplicate ticket IDs across splits; (4) threshold / abstain for low-max-prob tickets before routing to humans.

Lab Checklist

01

Plot the label histogram and write down the majority-class accuracy baseline

02

Train a tiny head-only classifier, then compare macro-F1 vs accuracy on the same val set

03

Build a 3×3 confusion matrix and identify the worst-recall class

04

Sweep a binary threshold (or max-prob abstain) and record precision/recall at two operating points

05

Induce label noise on 5% of train labels and measure how much val F1 drops

06

Export the best-val checkpoint and re-run inference with eval() / no-grad only

07

Document which metric you would ship on (accuracy vs macro-F1) and why

Pitfallswatch

Easy ways to fool yourself

metric

Wrong metric on skew

Report F1 / per-class recall — not only accuracy. Macro-F1 ≠ micro-F1.

leak

Train / val leakage

Same review in both splits → fake high scores.

noise

Label noise

Inconsistent human labels cap how good any model can get.

threshold

Threshold blindness

Binary default 0.5 may be wrong — tune on validation.
