Codecompile + fit

Skeleton: Sequential → compile → fit

model = keras.Sequential([ keras.layers.Dense(64, activation="relu"), keras.layers.Dense(NUM_CLASSES, activation="softmax"), ]) model.compile(optimizer="adam", loss="sparse_categorical_crossentropy", metrics=["accuracy"]) model.fit(x_train, y_train, epochs=EPOCHS, validation_data=(x_val, y_val), callbacks=[EarlyStopping(patience=3, restore_best_weights=True), ModelCheckpoint("best.keras", save_best_only=True)])

What one fit run is doing

stack

Small dense net

Ends in softmax with NUM_CLASSES units.

epoch

Each epoch

Shuffles batches, computes CE vs integer labels, updates weights.

stop

EarlyStopping

When val_loss bottoms out, restore that checkpoint — ready for inference.

Easy ways to go wrong

labels

Wrong CE variant

One-hot vs integer — mixing them fails silently or loudly.

val

No validation_data

You only see train accuracy and miss overfitting.

last

No EarlyStopping

Last epoch may be worse than an earlier one.

viz

Projector without labels

Color points by class — otherwise the cloud is hard to read.
