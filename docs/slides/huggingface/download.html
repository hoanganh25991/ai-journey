Deeper divenotes

Deeper dive

Trainer

Trainer vs raw loop

Trainer ≈ Keras fit for Transformers (device, eval, Hub upload). Raw loop wins for custom sampling / multi-loss.

LoRA

Full vs PEFT

Full: all ~110M BERT params. LoRA (r=8, α=16): <1% trainable — prefer when data is small or many task variants.

LR

Rates that work

Encoder: 2e-5–5e-5 + AdamW warmup. Head-only can use 1e-3 on the head. Too high on the stack → catastrophic forgetting.

11Decision guide
Case Studylab

Case Study

Ship a binary IMDb sentiment classifier from Hub assets to a Gradio Space in one afternoon.

inputs

Inputs

load_dataset("imdb") (~25k train), bert-base-uncased tokenizer + AutoModelForSequenceClassification(..., num_labels=2), max_length=256.

steps

Steps

map tokenize with padding/truncation → TrainingArguments(lr=2e-5, batch=16, epochs=3, eval each epoch, load_best_model_at_end) → Trainer.train() on a free GPU → push_to_hub("you/imdb-clf") → Space wraps pipeline("sentiment-analysis", model="you/imdb-clf").

output

Output

val accuracy typically mid–high 80s to ~90%; Hub repo contains config.json, weights, and tokenizer; Space URL is the shareable demo.

check

Verify

model card license allows your use; tokenizer files are present beside weights; private holdout (not only Hub test peek); Space cold-start still returns a label after wake.

13notes/huggingface.md
Pitfallswatch

Easy ways to go wrong

card

Ignoring the model card

Wrong license or unsafe intended use — especially commercial. Gated models need login + terms.

tok

Tokenizer mismatch

Different vocab than the checkpoint corrupts every ID.

lora

Full fine-tune when LoRA would do

Wasted GPU and money for small tasks.

eval

Peeking at Hub test

Keep a private holdout after exploring public splits. Wrong num_labels silently mis-shapes the head.
