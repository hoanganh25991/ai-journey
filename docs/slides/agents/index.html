06four axes
Cursorrouting

Auto vs fixed model

auto

Cursor Auto

Fast feedback loop — but quality varies with routing. Convenience can hide regressions.

fixed

Fixed (e.g. grok-4.5)

Controllable. In practice often better than Auto without burning much budget.

Worth investigating: token efficiency? routing? — treat Auto as a mode, not a model.

07Auto vs fixed
Exampleworked

Same bug, different arms

rich harness

Cursor + MCP + skills

“Tests fail on CI” → run tests, read logs, patch, re-run. Arms that act.

thin harness

No reliable shell/MCP

Same brain only suggests commands for you to paste. Arms that advise.

Don’t blame the model for missing tools — that is a harness gap.
