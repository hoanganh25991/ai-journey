13notes/skills-rules.md
Case Studylab

Case Study

Keep AI Journey notes English and structured without pasting the same instructions every chat.

inputs

Inputs

project glob rule on notes/**; global skills in ~/.agents/skills (frontend-slides, graphify, …); occasional /graphify . command.

steps

Steps

author a short glob rule (section order, English-only, don’t invent slide assets) → tighten skill descriptions with positive + negative triggers → ask “build a vector-DB deck” and confirm only frontend-slides loads → use /graphify . when you want an intentional graph rebuild.

output

Output

agents follow notes conventions when editing markdown; slide requests don’t fight a vague “docs” skill; graph rebuilds are explicit and rare.

check

Verify

always-rule token size; skill description false positives in real chats; no duplicate trees under ~/.cursor/skills; secrets never pasted into rules.
