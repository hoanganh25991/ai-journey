Why Grok 4.5 impresses

speed

~80 TPS

Optimized for coding + agentic work (xAI / mid-2026 context).

tokens

~4.2× fewer

Fewer output tokens than Opus 4.8 on SWE-Bench Pro → cheaper per task.

edge

Intel / time / $

Main edge = intelligence per unit time and cost on verifiable tasks.

Snapshot: ~$2 in / $6 out per 1M · trained with Cursor · often beats Auto. Watch overconfidence when you cannot run tests.

Deeper divenotes

Deeper dive · economics & gates

fit

Task shape > leaderboard

SWE-bench winners can still miss greenfield product sense. Keep a personal matrix; update when surprised twice.

$

Token economics

A “cheap” model that writes 4× tokens can cost more. Grok’s edge is partly output brevity under agent loops.

halluc

Hallucination surface

Unverifiable APIs punish confident models. Force tool use or pick ask-back Anthropic when stakes are high.

vision

Vision is a hard gate

Image attached → filter list first. Don’t route to DeepSeek and hope.

pair

Pair with harness

Strong model + tool-poor harness loses to mid model + shell/MCP. Read agents axes first.

fresh

Re-benchmark

Same failing test / scaffold quarterly. Field notes rot when vendors ship silent updates. Prices move.

11Decision guide
Case Studylab

Case Study

Route five real prompts without defaulting to “always Opus.”

inputs

Inputs

prompts A–E from the worked table (scaffold site, failing pytest, UI screenshot, skill draft, rename prop).

steps

Steps

A → Grok/Anthropic (greenfield); B → DeepSeek/Grok (debug in-tree); C → vision model only; D → Anthropic ask-back; E → Auto/Grok cheap edit. After each run, decide whether to change harness context before changing model.

output

Output

lower spend on mechanical edits; fewer blind UI guesses; skill design gets clarifying questions.

check

Verify

image gate before model pick; tools available on debug tasks; token volume not just $/1M sticker; quarterly re-run of A/B prompts after vendor updates.
